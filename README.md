# playwright-lovable-n8n

![CI](https://github.com/pulidoff/playwright-lovable-n8n/actions/workflows/ci.yml/badge.svg)

End-to-end test suite for a task manager app built with [Lovable](https://lovable.dev), automated with Playwright.

## Description

This project covers authentication and task management flows using the Page Object Model pattern. Tests run in CI on every push and pull request to `main`.

Authentication is handled via Playwright's `storageState` — you log in once manually and the session is reused across all tests, bypassing the Cloudflare Turnstile CAPTCHA present in the Lovable auth page.

## Setup

```bash
# Install dependencies
npm install

# Install Playwright browsers
npx playwright install --with-deps chromium
```

### Generate the auth session (required before first test run)

Because the Lovable auth page uses a Cloudflare Turnstile CAPTCHA that cannot be solved by headless browsers, you must log in manually once to generate a saved session:

```bash
npm run setup:auth
```

This opens a **headed** Chromium window at the `/auth` page. You have **60 seconds** to complete the login. Once the countdown finishes, the session is saved to `auth/storageState.json` and all subsequent test runs will reuse it without going through the login UI.

> `auth/storageState.json` is git-ignored — never commit it. Regenerate it when your session expires.

## Commands

```bash
# Generate auth session (run once before tests)
npm run setup:auth

# Run all tests (headless)
npm test

# Run tests in headed mode
npm run test:headed

# Open Playwright UI mode
npm run test:ui

# Show last HTML report
npm run report
```

## CI

Tests run automatically on push and pull requests to `main`. The CI pipeline requires a pre-generated `storageState.json` to be available. Store the contents of `auth/storageState.json` as a GitHub Actions secret named `STORAGE_STATE` and configure the workflow to write it to `auth/storageState.json` before running tests.
